# Skill-insight Optimizer

> **分类**: Skill 优化 | **成熟度**: 🟢 成熟期 | **综合评分**: 0.77

---

## 一句话描述

Skill-insight Optimizer 是 openEuler 社区推出的开源 Skill **全生命周期管理**工具中的优化模块，覆盖**"生成→评测→优化"闭环**。以执行过程数据驱动 Skill 进化：**语义聚合消除冗余**、**多维评测与过程追溯**、**动静态结合自优化**实现自动修复，填补了 skill-creator 在去冗余和过程追溯侧的空白。

## 核心实现

Skill-insight Optimizer 围绕三大核心能力构建：

**1. Skill 自动生成：Router + Scenario 架构**

采用轻量两级路由架构：
- **Router 层**：入口 SKILL.md 对用户 Prompt 进行意图嗅探和触发词匹配，精准分发到对应场景工作流，并强制注入统一输出模板规范
- **Scenario 层**：三大独立场景模块——**General**（自然语言主题驱动，5 步工作流：理解→调研→设计→确认→生成）、**Fault-Diagnosis**（故障溯源驱动，4 步排障工序，强制输出 `_lib.sh` 引擎库 + `collect.sh` 探针 + 独立排查脚本）、**Doc-Pipeline**（文档降维处理，支持单文档转 Skill、Merge Mode 多文档归纳、工程级批处理三种模式）
- **质量保障**：`validate_skill.sh` 实施 L1 结构校验 + L2 语法扫描，统一模板强制元数据标准化、组件结构固定化、脚本只读校验

**2. 多维深度评测：过程级可追溯**

构建三大核心观测维度：
- **执行精准度**：提取并校验 Agent 实际调用的 API 序列和命令行工具流，判定操作是否准确、是否存在冗余步骤
- **端到端时效**：测算从任务注入到最终输出的完整耗时，覆盖上下文加载、多轮思维链交互等各阶段
- **计算成本**：精确记录 Skill 加载时对大模型上下文窗口的占用量，引入 CPSR（每次成功解决成本）综合衡量准确率与成本的平衡

**3. 数据驱动自优化：动静态结合**

- **静态检查引擎**：Skill 运行前，结合 Linter 硬规则校验（格式、高危指令扫描）与大模型软规则评估（六维度质量评分：职责明确性、结构规范性、指令适配性、内容一致性、风险可控性、脚本质量），不达标则自动修复
- **动态反思引擎**：Skill 运行失败后，底层采集探针截获系统级报错信息，生成结构化缺陷诊断报告，仅锁定失效代码块进行定点修补（局部修补而非全量重写）
- **四种原子优化模式**：`static`（冷启动修复）、`dynamic`（基于报错轨迹修补）、`feedback`（用户反馈驱动迭代）、`hybrid`（先静态后动态组合），支持顺序编排
- **版本管理**：snapshots 目录 + meta.json 记录优化来源与模式，大/小版本号策略保障追溯完整性，Diff 视图 + 交互式确认闭环确保人在回路

---

## 主要能力

- **语义聚合式 Skill 生成**：通过去冗余、合相似、抽模式，将大量同类文档归纳为少量可复用 Skill，召回率从 30% 提升至 100%，Token 消耗降低 23%
- **多维评测与靶向归因**：覆盖准确率、时延、Token 成本、CPSR 四维指标，自动归因区分"模型缺陷"与"Skill 缺陷"，生成动态执行流程图对比预期与偏差
- **动静态结合的自优化闭环**：基于执行过程数据自动定位并修补 Skill 缺陷，支持交互式确认与细粒度版本回溯

## 局限性

- 目前主要支持 OpenAI SDK 兼容的 Agent
- 部分高级功能（Skill 质量评分、相似 Skill 聚类）在 Roadmap 中
