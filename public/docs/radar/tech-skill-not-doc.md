# Skill Is Not Document (R3)

## 一句话描述
提出Skill检索≠文档检索的核心论点，构建首个查询条件化的Skill路由基准R3-Skill，设计两阶段检索器（R3-Embedding+R3-Reranker），将LLM拒绝信号转化为Skill兼容性训练资源。

## 核心实现
- **核心论点**：传统文档检索的语义相关性无法保证Skill组合的正确性——多个Skill单独看都相关，但组合在一起可能无法协作完成任务（"Skill兼容性"问题）
- **Reject-as-Resource**：LLM在数据合成中的拒绝决策（哪些Skill不应被同时检索）通常被丢弃为低质量数据，R3将其转化为显式训练信号
- **两阶段检索**：R3-Embedding（双编码器）+ R3-Reranker（交叉编码器），梯度分析表明"推开"信号在双编码器中被双边平衡稀释，但在交叉编码器中作为无损分级排序监督——因此兼容性信号只在Reranker阶段使用
- **R3-Skill基准**：双语（中英）Skill检索基准，覆盖4个语言方向，查询表述接近真实用户请求，多专家交叉验证

## 主要能力
- Hit@1 = 0.7714，NDCG@10 = 0.8327，Set-Compat = 0.3525
- 首个面向Skill路由的查询条件化基准
- 证明Skill检索需要考虑组合兼容性，而非仅看个体相关性
- 开源数据集、训练代码和模型权重

## 局限性
- 基准规模和领域覆盖有限（仅4个语言方向）
- 仅评估检索质量，未端到端评估Agent任务完成率
- "Skill兼容性"定义依赖LLM合成数据，可能引入偏差
