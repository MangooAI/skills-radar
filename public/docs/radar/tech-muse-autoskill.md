# MUSE-Autoskill

> **分类**: Skill 生成 | **成熟度**: 🟡 成长期 | **综合评分**: 0.61

---

## 一句话描述

MUSE-Autoskill 提出五阶段统一技能生命周期（创建、记忆、管理、评估、改进），Agent 在任务执行中实时生成技能并通过单元测试验证入库，自生成技能在 SkillsBench 上准确率达 **87.94%**，超越人类专家编写的 68.40%，且跨 Agent 迁移零损耗。

## 核心实现

MUSE-Autoskill 将技能从"用完即扔的耗材"重构为"可维护、可测试、可迁移的知识资产"，核心是五阶段统一生命周期：

**1. 创建（Skill Creation）**：Agent 在 ReAct 循环内通过内置 skill_create 工具实时生成技能，同时产出 SKILL.md、s/、tests/ 完整包，遵循 Anthropic Agent Skills 开放标准。技能不是离线批量生产，而是"按需现场造"。

**2. 评估（Skill Evaluation）**：技能创建后必须在沙箱中跑 tests/ 单元测试，只有全部通过才能注册进技能银行（Skill Bank）。失败则 Agent 调用 update_skill 修补代码循环直到通过——"造完即测，测完才存"的硬门槛。

**3. 记忆（Skill Memory）**：每个技能旁配一个 .memory.md 文件，记录该技能在历次任务中的经验（已知失败场景、输入格式坑、性能注意事项）。下次加载技能时经验一并注入上下文，Agent 不需重复踩坑。技能级记忆与全局长期记忆的粒度区分是关键设计。

**4. 管理（Skill Management）**：每次任务开始时将技能库目录注入系统提示，Agent 自行决定用哪个技能。冗余技能合并，长期失败或未使用技能删除，保持库精简可用。

**5. 改进（Skill Improvement）**：技能产出错误结果时 Agent 直接 patch 代码，再次验证，持续迭代。

**关键设计**

- 两级自适应压缩机制（L1 原地摘要 + L2 段合并）：对话历史组织为 DAG，压缩只影响活跃链，原始历史通过不可变指针链完整保留。Token 接近阈值时触发，保证长任务不撑爆窗口
- 技能越详细 Token 反而越省：自生成技能平均 326 行（人类 146 行的 2.2 倍），但详细程序性说明替代了大量探索性推理，中位 Token 用量降 20%，耗时降 37%
- 单条轨迹蒸馏局限：hvac-control 任务准确率从 80% 跌至 20%，因把任务特异性校准参数当通用知识编码。改进方向为多轨迹综合提炼

## 主要能力

- 自生成技能 87.94% 准确率，超越人类专家 68.40%（SkillsBench 51 任务，GPT-5.5 底座）
- 跨 Agent 零损耗迁移：MUSE 生成技能注入 Hermes，准确率从 47.89% 提升至 58.40%，抹平 79% 差距
- 已有生产落地：SkillMarket（终端用户技能创建流水线）、ArkClaw（技能检索+子 Agent 封装）、SkillHub（团队级技能生命周期托管服务）

---

## 局限性

- 自生成技能仅覆盖 68.6% 任务，需特定专有工具深度知识或重型数值计算的任务无法蒸馏
- 单条轨迹蒸馏容易把任务特异性细节当通用知识编码，需多轨迹综合改进
- 生成一个技能的一次性成本约 383K Token 和 164 秒，虽首次复用即回本但初始投入较大
