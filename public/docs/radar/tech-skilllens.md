# SkillLens

> **分类**: Skill 评测 | **成熟度**: 🟡 成长期 | **综合评分**: 0.59

---

## 一句话描述

SkillLens 首次系统性拆解 Skill 生命周期为三阶段（经验生成→技能提取→技能消费），提出提取效能（EE）和目标可进化性（TE）两个互补指标，揭示 25% 的 Skill 会导致负迁移且无法靠文本判断优劣。

**来源**:
- 学术论文：微软研究院 + 上海交大 + 同济 + 复旦
- 发布年份：2026年

**链接**:
- 论文链接：https://arxiv.org/abs/2605.23899

## 核心实现

SkillLens 将 Skill 从"一段指令文本"重构为三阶段生命周期链路，每阶段有独立角色和评估维度：

**阶段 1：经验生成（Experience Generation）**
目标模型 M 在训练任务上执行，产生一批执行轨迹。这是 Skill 的原材料。

**阶段 2：技能提取（Skill Extraction）**
提取器模型 E 分析执行轨迹，蒸馏成结构化的 Skill 文档。提取器负责从经验中提炼知识。

**阶段 3：技能消费（Skill Consumption）**
同一个目标模型 M 带着提取出的 Skill 去做新任务，验证 Skill 是否真正提升表现。

**关键洞察**：提取器和目标模型是两个独立角色，性能不相关。一个提取能力强的模型可能是个弱消费者，反之亦然。

**两个核心指标**

- **提取效能（Extraction Efficacy, EE）**：固定一个提取器，它能为多少个不同目标模型稳定生产有用的 Skill？衡量的是 Skill 的泛化性
- **目标可进化性（Target Evolvability, TE）**：固定一个目标模型，当用不同提取器蒸馏经验时，它能获得多大提升？衡量的是模型吸收 Skill 的能力

**核心发现**

- 25% 的 Skill 会导致负迁移——成功率反而比没有 Skill 时更低
- 无法靠阅读 Skill 文本判断哪个更好，必须通过系统性评测
- 手动写的 Skill 三周后可能退化，系统性评测和自训练机制缺一不可

---

## 主要能力

- 首个系统性 Skill 生命周期评测框架
- EE/TE 双指标量化提取器泛化性和目标模型可进化性
- 揭示 Skill 负迁移现象（25%），为 Skill 质量门控提供理论依据
- 与 SkillOpt 配套：SkillLens 诊断问题，SkillOpt 自动修复

---

## 局限性

- 仅覆盖 Skill 生命周期的评测维度，不提供自动化优化方案（需搭配 SkillOpt）
- EE/TE 指标计算成本较高，需多模型多任务组合评测
- 论文阶段，无公开代码/工具
